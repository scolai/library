I built a code block to render any image. but I don't want front matter to provide image settings like path, base, alt, etc. I need each image to be accessable site-wide from a single location that will contain all parameters alongside the image. this sounds like a job for a data file.

This example is working with the logos which will appear on the site. And these can be company logos, and any other logo.

1. create a data file _data/logos.yml

header:
  path: /assets/images/logos
  base: header-logo
  alt: "Company header logo"
  has_avif: true
  has_webp: false

footer:
  path: /assets/images/logos
  base: footer-logo
  alt: "Company footer logo"
  has_avif: true
  has_webp: false

2. create the code that will work on rendering the logo _utilities/responsive-logo.html

{% assign path = include.path %}
{% assign base = include.base %}
{% assign alt = include.alt %}
{% assign caption = include.caption %}
{% assign widths = include.widths | default: "75,120" | split: "," %}

<!-- PNG -->
<img
	alt="{{ alt }}"
	src="{{ path }}/{{ base }}-75.png"
	srcset="{% for w in widths %}{{ path }}/{{ base }}-{{ w }}.png {{ w }}w{% unless forloop.last %},{% endunless %}{% endfor %}"
	sizes="(max-width: 600px) 75px, 120px">

3. insert include call where logos will appear eg _site-header
<picture class="header-logo"><a href="/">
  {% assign logo = site.data.logos.header %}
  {% include utilities/responsive-logo.html 
     path=logo.path
     base=logo.base
     alt=logo.alt 
     has_avif=logo.has_avif 
     has_webp=logo.has_avif %}</a><a href="/" class="logo-text">ビーバー達人ガイド</a>
</picture>

NOTE:
the logo's img tag in responsive-logo.html may require styling.
