Yes — `include` in `site-card.html` refers to the parameters passed in from `site-cards.html`. When you write:

```liquid
{% include site-card.html card=card %}
```

the loop variable `card` from `site-cards` is handed into `site-card`. Inside `site-card.html`, you access it as `include.card`. That’s why you can check `include.card.type` — it’s reading the `type` property of the object passed down from `site-cards`.  

Would that be best at this time?


Think of it this way:

site-cards.html: decides which dataset to loop over (testimonials, tours, etc.). That’s its only job — pick the source.

site-card.html: doesn’t “know” which dataset was chosen. It just receives one object at a time (include.card).

So the “block of code” question:

site-cards doesn’t grab different blocks from site-card. It always calls the same include.

Inside site-card, you branch on include.card.type (or another property) to decide which markup to render.

In other words:

site-cards = which cards to send.

site-card = how each card is rendered, depending on its type.

That’s the division of responsibility. site-cards doesn’t select blocks inside site-card; it just feeds the right data, and site-card itself chooses the markup.
